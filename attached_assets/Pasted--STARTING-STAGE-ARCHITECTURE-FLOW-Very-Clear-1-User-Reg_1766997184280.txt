
🔹 STARTING STAGE ARCHITECTURE FLOW (Very Clear)
1️⃣ User Registration (First Time)
🖥️ Frontend (Website / App)
User opens CampusConnect

Clicks Register

Enters:

Name

College email / ID

Role (Student / Faculty)

Branch, Year (for students)

Password

👉 Frontend only collects data, nothing is stored here.

🔁 Frontend → Backend (API Call)
Frontend sends data to backend:

POST /api/auth/register
Data sent as JSON:

{
  name,
  email,
  password,
  role,
  branch,
  year
}
⚙️ Backend (Server Logic)
Backend does:

Validates input

Hashes password

Creates unique User ID

Assigns role & permissions

Creates an empty AI profile for this user

🗄️ Database (Storage)
Backend stores:

User table:

user_id

name

email

role

branch

year

AI_Profile table:

user_id

learning_history = empty

mood_history = empty

preferences = default

👉 This is how AI Mentor becomes user‑specific from day one

2️⃣ User Login (Every Time)
🖥️ Frontend
User enters email + password

Clicks Login

🔁 Frontend → Backend
POST /api/auth/login
⚙️ Backend
Verifies credentials

Generates JWT token

Sends token + basic profile info

{
  token,
  user_id,
  role,
  name
}
🖥️ Frontend
Stores token securely

Loads dashboard

Uses token for every future request

👉 Frontend never directly talks to database or AI.

3️⃣ Fetching User Data After Login
🖥️ Frontend
Requests dashboard data:

GET /api/dashboard
Authorization: Bearer <token>
⚙️ Backend
Reads token

Identifies exact user

Fetches:

Profile data

AI profile data

Sends data to frontend

4️⃣ AI Mentor Association (AWS – Simple)
🤖 Where AI Lives
AI Mentor runs as:

AWS Lambda / EC2

Uses NLP + ML models

AI does NOT know users directly

🔁 Backend → AWS AI Mentor
When needed (even at start):

POST /ai/mentor/analyze
{
  user_id,
  data
}
🤖 AI Mentor (AWS)
Uses user_id as key

Reads only that user’s data

Generates insights

Sends result back to backend

⚙️ Backend
Stores AI output linked to same user_id

Sends response to frontend

🖥️ Frontend
Shows AI Mentor message:

Welcome guidance

Initial suggestions

👉 One user = one AI context
👉 No data mixing between users

5️⃣ Key Point (Very Important for Guides)
Frontend = UI only

Backend = control + security

Database = storage

AI (AWS) = intelligence

User ID is the link between everything
